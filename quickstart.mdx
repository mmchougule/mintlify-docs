This guide walks you through installing the SDK, creating a `B402Solana` instance, shielding USDC into the pool, and executing a private swap. By the end you will have a working TypeScript snippet that runs against the b402 mainnet pool using the hosted relayer — no relayer configuration required.

## Prerequisites

Before you start, make sure you have:

- **Node.js 20 or later** — the SDK uses native `crypto.getRandomValues`, available in Node 18+ but best supported in Node 20.
- **A Solana keypair** — the default path is `~/.config/solana/id.json`. This wallet signs the initial shield transaction only.
- **USDC or wSOL on mainnet** — the two mints whitelisted in the current pool deployment. You need at least the amount you plan to shield.
- **A private RPC URL** — the public `api.mainnet-beta.solana.com` endpoint rate-limits aggressively. Use a provider such as Helius, Triton, QuickNode, or Alchemy.

<Warning>
  Never commit your keypair file or RPC URL to version control. Use environment variables for both.
</Warning>

## Steps

<Steps>
  <Step title="Install the SDK">
    Add `@b402ai/solana` to your project.

    ```bash
    npm install @b402ai/solana
    ```
  </Step>

  <Step title="Import and construct B402Solana">
    Create a `B402Solana` instance with your cluster, RPC URL, and keypair. The hosted relayer URL is applied automatically for `mainnet` and `devnet` — you do not need to set `relayerHttpUrl` unless you want to override it.

    ```typescript
    import { Keypair, PublicKey } from '@solana/web3.js';
    import { B402Solana } from '@b402ai/solana';

    const b402 = new B402Solana({
      cluster: 'mainnet',
      rpcUrl: process.env.RPC_URL!,   // Helius / Triton / your own
      keypair: yourKeypair,            // user signs the initial shield only
    });
    ```
  </Step>

  <Step title="Call ready()">
    `ready()` fetches the Groth16 circuit artifacts (~36 MB) on first call, SHA256-verifies them, and caches them at `~/.b402ai/circuits`. Subsequent calls return immediately from cache.

    ```typescript
    await b402.ready();
    ```
  </Step>

  <Step title="Shield tokens">
    Move public USDC into the b402 shielded pool. Your wallet signs this one transaction. After this point, no further transaction references your wallet.

    ```typescript
    const USDC = new PublicKey('EPjFWdd5AufqSSqeM2qN1xzybapC8G4wEGGkZwyTDt1v');

    await b402.shield({ mint: USDC, amount: 1_000_000n }); // 1 USDC (6 decimals)
    ```

    The deposit amount is visible on-chain. The link between your wallet and any subsequent spending is not.
  </Step>

  <Step title="Execute a private swap">
    Call `swap()` to exchange the shielded USDC for wSOL via Jupiter. The hosted relayer signs and pays gas. Your wallet does not appear on this transaction.

    ```typescript
    const SOL = new PublicKey('So11111111111111111111111111111111111111112');

    const { signature, outAmount } = await b402.swap({
      inMint: USDC,
      outMint: SOL,
      amount: 1_000_000n,
    });

    console.log('swap finalized:', signature);
    console.log('received (lamports):', outAmount.toString());
    ```
  </Step>
</Steps>

## Complete example

The following is the full working snippet from [`docs/getting-started.md`](https://github.com/mmchougule/b402-solana/blob/main/docs/getting-started.md):

```typescript
import { Keypair, PublicKey } from '@solana/web3.js';
import { B402Solana } from '@b402ai/solana';

const b402 = new B402Solana({
  cluster: 'mainnet',
  rpcUrl: process.env.RPC_URL!,        // Helius / Triton / your own
  keypair: yourKeypair,                 // user signs the initial shield only
});
await b402.ready();   // first call fetches circuit artifacts (~36MB,
                       // SHA256-verified, cached at ~/.b402ai/circuits)

const USDC = new PublicKey('EPjFWdd5AufqSSqeM2qN1xzybapC8G4wEGGkZwyTDt1v');
const SOL  = new PublicKey('So11111111111111111111111111111111111111112');

// Move public USDC into the shielded pool (one-time, user signs).
await b402.shield({ mint: USDC, amount: 1_000_000n });

// Private swap — relayer signs + pays gas. Caller wallet stays off-chain.
const { signature, outAmount } = await b402.swap({
  inMint: USDC,
  outMint: SOL,
  amount: 1_000_000n,
});
```

You can run the full quickstart script directly from the repo:

```bash
git clone https://github.com/mmchougule/b402-solana
cd b402-solana
pnpm install
B402_RPC_URL='https://mainnet.helius-rpc.com/?api-key=...' \
  pnpm exec node --experimental-strip-types examples/quickstart-private-swap.ts
```

## What just happened

Understanding the two operations helps you reason about the privacy model.

**`shield`** moves your tokens from a public wallet into the b402 pool. The pool records a Poseidon commitment to a new UTXO note in its on-chain Merkle tree. Your wallet signs this transaction, so the deposit amount is linked to your address. After this, no further transaction references your wallet.

**`swap`** builds an adapter instruction that binds a Groth16 proof to the Jupiter adapter ID, the route hash, and the expected output. The pool's `adapt_execute` instruction verifies the proof on-chain, calls the Jupiter adapter via CPI, and mints a new shielded note for the output amount. The b402 hosted relayer submits this transaction — your wallet does not appear.

### What is visible vs hidden

| Visible on-chain | Hidden |
|---|---|
| Amount of the swap | Who deposited |
| Adapter program (Jupiter) | Who receives the output note |
| Fill venue (Phoenix, Raydium, …) | Link between USDC deposit and SOL output |
| Relayer pubkey (paid the gas) | Your wallet address |
