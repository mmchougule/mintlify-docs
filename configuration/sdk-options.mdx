---
title: "B402Solana constructor options complete reference"
description: "Complete reference for every B402Solana constructor option including cluster, signer, RPC URL, relayer, note persistence, and indexer settings."
---

The `B402SolanaConfig` interface controls every aspect of how the SDK connects to Solana, signs transactions, routes through the relayer, persists note state between process restarts, and fetches Merkle proofs. You pass this object as the single argument to `new B402Solana(...)`. Most fields are optional — a minimal mainnet configuration needs only `cluster`, `keypair`, and optionally `rpcUrl`.

## Constructor options

<ParamField path="cluster" type="'mainnet' | 'devnet' | 'localnet'" required>
  The Solana cluster to connect to. Controls which hosted relayer and indexer URLs are used by default, which program IDs are targeted, and whether `inlineCpiNullifier` defaults to `true` (mainnet and devnet) or `false` (localnet). See [Environments](/configuration/environments) for a full comparison.
</ParamField>

<ParamField path="keypair" type="Keypair">
  A `@solana/web3.js` `Keypair` that owns the shielded notes. Use this for Node.js backends, MCP server integrations, and tests. The SDK wraps it as a `KeypairSigner` internally. Pass `keypair` **or** `signer` — not both.
</ParamField>

<ParamField path="signer" type="B402Signer">
  A `B402Signer` abstraction for browser environments where you cannot access the raw private key bytes. On-chain signing routes through `signer.signTransaction()`; the wallet seed is derived from `signer.getSeed()`. Use `WalletAdapterSigner` (re-exported from `@b402ai/solana`) to wrap any Solana wallet adapter. Pass `signer` **or** `keypair` — not both.
</ParamField>

<ParamField path="rpcUrl" type="string">
  A full HTTP or HTTPS Solana RPC endpoint. Defaults to the cluster's public endpoint (`api.mainnet-beta.solana.com` or `api.devnet.solana.com`), which is rate-limited and unsuitable for production. Use a private RPC from Helius, Triton, QuickNode, or Alchemy for any real traffic.
</ParamField>

<ParamField path="relayer" type="Keypair">
  A `Keypair` to use as the fee payer when submitting transactions yourself instead of using the hosted relayer. Defaults to `keypair` when omitted, so in single-key development setups the same key shields and pays gas. In production, prefer `relayerHttpUrl` so your user's wallet is never the fee payer on unshield or swap transactions.
</ParamField>

<ParamField path="relayerHttpUrl" type="string">
  The base URL of an HTTP relayer. When set, `unshield`, `swap`, and other operations that require no user signature are routed through the relayer — the on-chain fee payer becomes the relayer's wallet, keeping your user's wallet off those transactions. `shield` still signs locally because Anchor requires the depositor as a signer for the SPL transfer. Calling `ready()` sends a `/health` request to discover the relayer's public key. Defaults to b402's hosted relayer for mainnet and devnet:
  - Mainnet: `https://b402-solana-relayer-mainnet-62092339396.us-central1.run.app`
  - Devnet: `https://b402-solana-relayer-devnet-62092339396.us-central1.run.app`
</ParamField>

<ParamField path="relayerApiKey" type="string">
  An API key forwarded as `Authorization: Bearer <key>` to the relayer on every request. The public tier key is baked into the SDK and allows 5 requests per minute. Override this field with a private key for production integrations. Contact the b402 team to request a production key.
</ParamField>

<ParamField path="notesPersistDir" type="string">
  A directory path where the SDK writes note state to disk. Each viewing key gets its own file: `<dir>/<viewingPubHex>.json`, so multiple wallets can coexist in the same directory. Without this option, note state is lost when the process exits, requiring a full re-scan of the chain on startup. Defaults to `~/.b402ai/notes/<cluster>`.

  <Warning>
    Note files are stored in plaintext. Their threat model is the same as your Solana keypair file — protect the directory with appropriate filesystem permissions.
  </Warning>
</ParamField>

<ParamField path="indexerUrl" type="string">
  The base URL of a Merkle proof oracle. When set, `unshield` and `swap` fetch Merkle inclusion proofs for any leaf index via `GET /v1/proof?leafIndex=N`. This removes the rightmost-only restriction of `proveMostRecentLeaf`, enabling multi-deposit unshield in any order and cross-device note discovery.

  The SDK always verifies the indexer's claimed Merkle root against the on-chain `TreeState` before using any proof — a compromised indexer can cause a denial of service but cannot produce a forgeable spend. When the indexer is unreachable, the SDK falls back to `proveMostRecentLeaf` automatically.

  Defaults to b402's hosted indexer on mainnet: `https://b402-solana-indexer-api-62092339396.us-central1.run.app`. There is no hosted devnet indexer yet.
</ParamField>

<ParamField path="inlineCpiNullifier" type="boolean" default="true">
  When `true`, unshield and swap transactions encode the nullifier as a CPI call from the pool rather than as a sibling instruction, saving approximately 150 wire bytes per nullifier. Defaults to `true` for mainnet and devnet. Only set this to `false` when testing against a custom localnet deployment.
</ParamField>

<ParamField path="proverArtifacts" type="ProverArtifacts">
  Paths to the transact circuit `.wasm` and `.zkey` files used to generate Groth16 proofs for `shield` and `unshield`. If omitted and no `prover` is supplied, the SDK fetches the artifacts automatically from the `circuits-v1` release tag on GitHub and SHA256-verifies them before use.

  ```typescript
  type ProverArtifacts = {
    wasmPath: string;
    zkeyPath: string;
  };
  ```
</ParamField>

<ParamField path="adaptProverArtifacts" type="ProverArtifacts">
  Paths to the adapt circuit `.wasm` and `.zkey` files used to generate Groth16 proofs for `swap` and `lend`. Required if `adaptProver` is not supplied. Uses the same `ProverArtifacts` shape as `proverArtifacts`. Auto-fetched from the `circuits-v1` release if omitted.
</ParamField>

## Examples

<CodeGroup>

```typescript Minimal mainnet config
import { B402Solana } from '@b402ai/solana';
import { Keypair } from '@solana/web3.js';

const keypair = Keypair.fromSecretKey(/* your key bytes */);

const b402 = new B402Solana({
  cluster: 'mainnet',
  keypair,
  rpcUrl: process.env.RPC_URL, // use a private RPC in production
});

await b402.ready();
```

```typescript Full production config
import { B402Solana } from '@b402ai/solana';
import { Keypair } from '@solana/web3.js';
import os from 'os';
import path from 'path';

const keypair = Keypair.fromSecretKey(/* your key bytes */);

const b402 = new B402Solana({
  cluster: 'mainnet',
  keypair,
  rpcUrl: process.env.RPC_URL,              // private Helius / Triton endpoint
  relayerHttpUrl:
    'https://b402-solana-relayer-mainnet-62092339396.us-central1.run.app',
  relayerApiKey: process.env.B402_API_KEY,  // production key; 5 req/min if omitted
  indexerUrl:
    'https://b402-solana-indexer-api-62092339396.us-central1.run.app',
  notesPersistDir: path.join(os.homedir(), '.b402ai', 'notes', 'mainnet'),
  inlineCpiNullifier: true,
});

await b402.ready();
```

</CodeGroup>

## Browser vs Node.js

In Node.js and MCP environments you pass `keypair` directly. In a browser you never have access to the raw private key, so use `WalletAdapterSigner` instead:

```typescript
import { B402Solana, WalletAdapterSigner } from '@b402ai/solana';
import { useWallet } from '@solana/wallet-adapter-react';

const wallet = useWallet();

const b402 = new B402Solana({
  cluster: 'mainnet',
  signer: new WalletAdapterSigner(wallet),
  rpcUrl: process.env.NEXT_PUBLIC_RPC_URL,
});
```

<Note>
  Never pass both `keypair` and `signer` to the constructor. The SDK will throw at initialization if both are present.
</Note>

<Tip>
  Call `await b402.ready()` immediately after construction. It fetches circuit artifacts (if not bundled), verifies their SHA256 hashes, and performs a `/health` check against the relayer to discover its public key. Operations will work without calling `ready()`, but the first call that needs an artifact will block while it downloads.
</Tip>
