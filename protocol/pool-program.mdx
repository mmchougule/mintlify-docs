`b402-pool` is the shielded pool. It owns custody, holds the merkle tree state, dispatches instructions, CPIs into the two verifier programs, CPIs `b402_nullifier` for double-spend gating, and CPIs registered adapters.

Source: [`programs/b402-pool/src/`](https://github.com/mmchougule/b402-solana/tree/main/programs/b402-pool/src). Program ID (devnet + mainnet): `42a3hsCXtQLWonyxWZosaaCJCweYYKMrvNd25p1Jrt2y` ([`lib.rs:25`](https://github.com/mmchougule/b402-solana/blob/main/programs/b402-pool/src/lib.rs#L25)).

The mainnet binary is built with `--features prd_35_pending_inputs,inline_cpi_nullifier,phase_9_dual_note`.

## `shield`

Source: [`shield.rs`](https://github.com/mmchougule/b402-solana/blob/main/programs/b402-pool/src/instructions/shield.rs). User deposits tokens, appends one or two commitments to the tree.

**Accounts** ([`shield.rs:51-99`](https://github.com/mmchougule/b402-solana/blob/main/programs/b402-pool/src/instructions/shield.rs#L51-L99)):

| # | Role | Notes |
|---|---|---|
| 0 | relayer (signer, mut) | Pays SOL fee |
| 1 | depositor (signer, mut) | Authorizes the SPL transfer |
| 2 | depositor_token_account (mut) | Source ATA |
| 3 | TokenConfig | Proves mint is whitelisted |
| 4 | Vault (mut) | Destination |
| 5 | TreeState (mut) | Tree leaf append |
| 6 | PoolConfig | Pause flag + pinned verifier addr |
| 7 | verifier_transact program | For CPI |
| 8 | spl-token program | For CPI |
| 9 | system program | For account creation |

**Args** ([`shield.rs:43-49`](https://github.com/mmchougule/b402-solana/blob/main/programs/b402-pool/src/instructions/shield.rs#L43-L49)):

```rust
pub struct ShieldArgs {
    pub proof: Vec<u8>,                    // must be 256 bytes
    pub public_inputs: TransactPublicInputs,
    pub encrypted_notes: Vec<EncryptedNote>, // 0..=2 entries
    pub note_dummy_mask: u8,
}
```

**Validation order** (handler [`shield.rs:101-227`](https://github.com/mmchougule/b402-solana/blob/main/programs/b402-pool/src/instructions/shield.rs#L101-L227)):

1. `proof.len() == 256`, `encrypted_notes.len() <= 2`.
2. `paused_shields == false`.
3. `verifier_program.key() == pool_config.verifier_transact`.
4. `public_amount_in > 0`, `public_amount_out == 0`, `relayer_fee == 0`.
5. `pi.public_token_mint == token_config.mint`.
6. `vault.amount + public_amount_in <= token_config.max_tvl`.
7. `nullifier[0] == nullifier[1] == 0` (shield has no inputs).
8. `merkle_root` is in the recent 128-root ring (`util::tree_has_recent_root`).
9. CPI `b402_verifier_transact::verify(proof, 18 LE inputs)`.
10. CPI `spl-token::transfer(depositor_ata → vault, public_amount_in)`.
11. For each non-dummy `commitment_out[i]`: `tree_append`, emit `CommitmentAppended`.
12. Emit `ShieldExecuted { mint, amount, slot }`.

CU range: ~240k. Tx size: ~1,157 B.

## `transact`

Source: [`transact.rs`](https://github.com/mmchougule/b402-solana/blob/main/programs/b402-pool/src/instructions/transact.rs). Spend N notes, create M notes, no public-amount movement.

```rust
pub struct TransactArgs {
    pub proof: Vec<u8>,
    pub public_inputs: TransactPublicInputs,
    pub encrypted_notes: Vec<EncryptedNote>,
    pub in_dummy_mask: u8,
    pub out_dummy_mask: u8,
    #[cfg(feature = "inline_cpi_nullifier")]
    pub nullifier_cpi_payloads: Vec<[u8; 134]>,
}
```

**Accounts**: relayer, PoolConfig, TreeState (mut), verifier_program, instructions_sysvar, spl-token, system. Plus `remaining_accounts` for inline-CPI nullifier inserts (10 accounts per real nullifier).

**Validation:**

1. Pause check (`paused_transacts`).
2. `public_amount_in == 0 && public_amount_out == 0`.
3. Root ring membership.
4. Verifier CPI (18 inputs).
5. `nullifier[0] < nullifier[1]` when both non-dummy (canonicalization, [`unshield.rs:199-204`](https://github.com/mmchougule/b402-solana/blob/main/programs/b402-pool/src/instructions/unshield.rs#L199-L204)).
6. Nullifier insert (see `inline_cpi_nullifier` section below).
7. Tree append for non-dummy commitments.

## `unshield`

Source: [`unshield.rs`](https://github.com/mmchougule/b402-solana/blob/main/programs/b402-pool/src/instructions/unshield.rs). Spend notes, withdraw to a clear recipient ATA. **Unshield is never paused** ([`unshield.rs:114`](https://github.com/mmchougule/b402-solana/blob/main/programs/b402-pool/src/instructions/unshield.rs#L114)).

**Validation order** ([`unshield.rs:103-381`](https://github.com/mmchougule/b402-solana/blob/main/programs/b402-pool/src/instructions/unshield.rs#L103-L381)):

1. `proof.len() == 256`, `encrypted_notes.len() <= 2`.
2. `verifier_program.key() == pool_config.verifier_transact`.
3. `public_amount_in == 0`, `public_amount_out > 0`, `relayer_fee <= public_amount_out`.
4. `pi.public_token_mint == token_config.mint`.
5. Root ring membership.
6. **Recipient binding** ([`unshield.rs:170-185`](https://github.com/mmchougule/b402-solana/blob/main/programs/b402-pool/src/instructions/unshield.rs#L170-L185)):
   ```
   pi.recipient_bind == Poseidon_3(
       TAG_RECIPIENT_BIND,
       owner.bytes[0..16] | low,
       owner.bytes[16..32] | high
   )
   ```
   Stops a relayer from substituting their own ATA — the proof was generated against `recipient_token_account.owner`, not against the account address itself.
7. Verifier CPI (18 inputs).
8. `nullifier[0] < nullifier[1]` when both non-dummy.
9. Nullifier inserts via the active mode (sibling-ix or inline CPI).
10. Tree append for non-dummy `commitment_out[i]`.
11. Pool-signed `vault → recipient_token_account` for `public_amount_out − relayer_fee`.
12. Pool-signed `vault → relayer_fee_token_account` when `relayer_fee > 0`.
13. Emit `UnshieldExecuted`.

CU range: ~350k.

## `adapt_execute`

Source: [`adapt_execute.rs`](https://github.com/mmchougule/b402-solana/blob/main/programs/b402-pool/src/instructions/adapt_execute.rs). Composable execution. Burns input notes in IN-mint, CPIs an adapter, mints output notes in OUT-mint, all bound by the `adapt` Groth16 proof.

### `AdaptPublicInputs` (current wire shape)

The struct in [`adapt_execute.rs:65-95`](https://github.com/mmchougule/b402-solana/blob/main/programs/b402-pool/src/instructions/adapt_execute.rs#L65-L95):

```rust
pub struct AdaptPublicInputs {
    pub merkle_root: [u8; 32],
    pub nullifier: [[u8; 32]; 2],
    pub commitment_out: [[u8; 32]; 2],
    pub public_amount_in: u64,
    pub public_amount_out: u64,        // adapt requires zero
    pub relayer_fee: u64,
    pub relayer_fee_bind: [u8; 32],
    pub root_bind: [u8; 32],
    pub recipient_bind: [u8; 32],
    pub expected_out_value: u64,
    #[cfg(feature = "phase_9_dual_note")]
    pub out_spending_pub: [u8; 32],    // outSpendingPub[0] alias from circuit

```

Four fields previously on the wire have been **dropped**: `public_token_mint`, `expected_out_mint`, `adapter_id`, `action_hash`. The pool reconstructs each from on-chain state during the verifier-input build:

| Removed | Reconstructed from | Where |
|---|---|---|
| `public_token_mint` | `token_config_in.mint` | [`adapt_execute.rs:1132-1134`](https://github.com/mmchougule/b402-solana/blob/main/programs/b402-pool/src/instructions/adapt_execute.rs#L1132-L1134) |
| `expected_out_mint` | `token_config_out.mint` | [`adapt_execute.rs:1152-1153`](https://github.com/mmchougule/b402-solana/blob/main/programs/b402-pool/src/instructions/adapt_execute.rs#L1152-L1153) |
| `adapter_id` | `keccak(adapter_program.key) mod p` | [`adapt_execute.rs:1146-1148`](https://github.com/mmchougule/b402-solana/blob/main/programs/b402-pool/src/instructions/adapt_execute.rs#L1146-L1148) |
| `action_hash` | `Poseidon_3(TAG_ADAPT_BIND, keccak(action_payload) mod p, out_mint Fr)` | [`adapt_execute.rs:426-441`](https://github.com/mmchougule/b402-solana/blob/main/programs/b402-pool/src/instructions/adapt_execute.rs#L426-L441) |

`out_spending_pub` is added under `phase_9_dual_note` so the pool can recompute the excess-output commitment in Rust ([`adapt_execute.rs:920-957`](https://github.com/mmchougule/b402-solana/blob/main/programs/b402-pool/src/instructions/adapt_execute.rs#L920-L957)). Net wire change is roughly neutral (−96 B from the four drops, +32 B from `out_spending_pub`).

### Validation order

From the handler ([`adapt_execute.rs:361-993`](https://github.com/mmchougule/b402-solana/blob/main/programs/b402-pool/src/instructions/adapt_execute.rs#L361-L993)):

1. `proof.len() == 256`; `raw_adapter_ix_data.len() >= 8`.
2. `paused_adapts == false`.
3. `verifier_program == pool_config.verifier_adapt`.
4. `public_amount_out == 0`, `public_amount_in > 0`.
5. `adapter_program` is in `AdapterRegistry`, `enabled`, and `raw_adapter_ix_data[0..8]` is in its discriminator allowlist.
6. Recompute `computed_action_hash` from `keccak(action_payload)` and `token_config_out.mint`.
7. Root ring membership.
8. Dummy-nullifier values are zero.
9. `relayer_fee <= public_amount_in`; if `fee > 0`, `relayer_fee_ta.owner == args.relayer_fee_recipient`.
10. Build the public-input vector via `build_public_inputs_for_adapt`, injecting reconstructed `in_mint`, `out_mint`, `adapter_id`, `action_hash`.
11. **Verifier path** (one of):
    - **PRD-35 (`prd_35_pending_inputs`)**: validate `pending_inputs` PDA is the canonical derivation from `pi.out_spending_pub`, byte-equality-check the on-chain inputs against the pool's recomputed vector, then CPI `verify_with_account_inputs(proof)`. Replay protection: zero `pending_inputs.version` post-verify ([`adapt_execute.rs:495-549`](https://github.com/mmchougule/b402-solana/blob/main/programs/b402-pool/src/instructions/adapt_execute.rs#L495-L549)).
    - **Phase 7B inline**: CPI `verify(proof, public_inputs)` ([`adapt_execute.rs:552-557`](https://github.com/mmchougule/b402-solana/blob/main/programs/b402-pool/src/instructions/adapt_execute.rs#L552-L557)).
12. `nullifier[0] < nullifier[1]` when both non-dummy.
13. Nullifier inserts (sibling-ix or inline-CPI mode).
14. **Input-vault transfer** — pool signs `in_vault → adapter_in_ta` for `public_amount_in` … unless the **stateful + synthetic-voucher** skip applies (see below).
15. Pool-signed relayer-fee transfer if `fee > 0`.
16. Snapshot `pre = out_vault.amount`.
17. CPI `adapter_program::execute` with the assembled `AccountMeta` list and the `cpi_ix_data` (action_payload prefixed with `viewing_pub_hash` for stateful adapters; verbatim for stateless).
18. Reload `out_vault`; compute `delta = post − pre`.
19. **I4 slippage gate** with the **stateful-voucher skip** ([`adapt_execute.rs:842-853`](https://github.com/mmchougule/b402-solana/blob/main/programs/b402-pool/src/instructions/adapt_execute.rs#L842-L853)).
20. Tree append for non-dummy `commitment_out[i]`.
21. **Phase 9 excess-note minting** when `delta > expected_out_value` (see below).
22. Emit `AdaptExecuted { adapter_program, in_mint, out_mint, public_amount_in, out_delta, expected_out_value, relayer_fee, slot }`.

CU range: ~600k–680k.

### Sub-section: stateful-adapter slippage skip (Path A)

Per [`adapt_execute.rs:809-853`](https://github.com/mmchougule/b402-solana/blob/main/programs/b402-pool/src/instructions/adapt_execute.rs#L809-L853): adapters that credit user claims into protocol state (e.g. Kamino's per-user `Obligation` owned by `owner_pda`) never deliver tokens to `out_vault`. The standard `delta >= expected_out_value` check is unsatisfiable for these. The pool gates with:

```rust
let stateful_voucher_mint = is_stateful_adapter(&adapter_program_key)
    && delta < pi.expected_out_value;
if !stateful_voucher_mint {
    require!(delta >= pi.expected_out_value, AdapterReturnedLessThanMin);
}
```

The voucher commitment (`pi.commitment_out[0]`) is appended unconditionally — the user's claim against the off-pool state is the proof-bound shielded note. On `privateRedeem`, USDC physically arrives in `out_vault` and the standard slippage check passes.

The stateful-adapter list is hardcoded ([`adapt_execute.rs:1035-1040`](https://github.com/mmchougule/b402-solana/blob/main/programs/b402-pool/src/instructions/adapt_execute.rs#L1035-L1040)). Adding one is a pool source change, not a registry flag.

### Sub-section: synthetic voucher mint handling

For `(stateful adapter, synthetic in_mint)` the input transfer `in_vault → adapter_in_ta` is **skipped** ([`adapt_execute.rs:683-705`](https://github.com/mmchougule/b402-solana/blob/main/programs/b402-pool/src/instructions/adapt_execute.rs#L683-L705)) — the voucher (e.g. kUSDC) was never backed by liquid tokens in the vault; the user's claim lives in the per-user obligation. The synthetic mint list is also hardcoded ([`adapt_execute.rs:1056-1063`](https://github.com/mmchougule/b402-solana/blob/main/programs/b402-pool/src/instructions/adapt_execute.rs#L1056-L1063)).

### Sub-section: `inline_cpi_nullifier` CPI to `b402-nullifier`

Under `inline_cpi_nullifier` ([`adapt_execute.rs:609-661`](https://github.com/mmchougule/b402-solana/blob/main/programs/b402-pool/src/instructions/adapt_execute.rs#L609-L661)), `remaining_accounts` is laid out:

```
[0]                              = b402_nullifier program AccountInfo
[1 .. 1 + 10·real_count]         = b402_nullifier accounts (10 per real nullifier)
[1 + 10·real_count .. ]          = adapter-specific remaining accounts
```

Per real input nullifier (10 accounts: payer, instructions sysvar, light_system_program, cpi_authority, registered_program_pda, account_compression_authority, account_compression_program, system_program, address_tree, output_queue), the pool calls `invoke_create_nullifier` ([`nullifier_cpi.rs:100-139`](https://github.com/mmchougule/b402-solana/blob/main/programs/b402-pool/src/instructions/nullifier_cpi.rs#L100-L139)) which builds the inner ix:

```
data:
  [0..8)    = sha256("global:create_nullifier")[..8]
  [8..137)  = ValidityProof Borsh
  [137..141) = PackedAddressTreeInfo Borsh
  [141..142) = output_state_tree_index
  [142..174) = id (the nullifier value, 32 bytes)
```

The 134-byte caller payload `args.nullifier_cpi_payloads[i]` is the `[8..142)` slice; the pool appends the 32-byte nullifier itself so the bound the proof committed to cannot be tampered with by the SDK.

### `adapt_execute_v2` (PRD-35 lean path)

Identical to `adapt_execute` minus `public_inputs` on the wire. The pool reads them from the `pending_inputs` PDA via `parse_adapt_public_inputs_from_pda` ([`adapt_execute.rs:336-358`](https://github.com/mmchougule/b402-solana/blob/main/programs/b402-pool/src/instructions/adapt_execute.rs#L336-L358)) and delegates to the shared handler. Saves ~320 B on the wire — the actual fix that lifts the v0-tx 1232 B ceiling for per-user adapters.

Writes the 24 × 32 B Phase-9 public-input vector into the per-user `PendingInputs` PDA at seeds `[b"b402/v1", b"pending-inputs", spending_pub_le]`. ## Admin instructions

Source: [`admin.rs`](https://github.com/mmchougule/b402-solana/blob/main/programs/b402-pool/src/instructions/admin.rs). The admin gate is a single `admin_multisig` pubkey ([`admin.rs:12-15`](https://github.com/mmchougule/b402-solana/blob/main/programs/b402-pool/src/instructions/admin.rs#L12-L15)) — signer must equal `pool_config.admin_multisig`. Multi-signer enforcement is not implemented.

| Instruction | Effect |
|---|---|
| `pause(which)` / `unpause(which)` | Sets `paused_shields` / `paused_transacts` / `paused_adapts`. Cannot pause unshield. |
| `set_verifier(kind, new_id)` | Swaps `verifier_transact` / `verifier_adapt` / `verifier_disclose`. |
| `register_adapter(info)` | Appends to `AdapterRegistry`. **No de-register** — the allowlist is append-only. |
| `set_max_tvl(new_max_tvl)` | Updates the per-mint cap. |
| `add_token_config(args)` | Whitelists an SPL mint. |
| `gc_pending_inputs` | Closes a stale `PendingInputs` PDA, rent → `treasury_pubkey`. |

There is no `disable_adapter`, `enable_adapter`, `queue_upgrade`, `revoke_upgrade_authority`, or `rotate_admin` instruction. Program upgrade authority is the standard Solana BPF authority, owned by the deployer.

## Security invariants (program-level)

From [`programs/b402-pool/src/lib.rs:7-13`](https://github.com/mmchougule/b402-solana/blob/main/programs/b402-pool/src/lib.rs#L7-L13) (the binding list, enforced across handlers):

1. No nullifier insertion without a successful verifier CPI prior in the same instruction.
2. No vault transfer without balance conservation enforced by the proof.
3. Unshield cannot be paused.
4. Adapt CPIs are followed by a post-call balance-delta check (subject to the stateful-voucher skip).

Additional handler-level invariants:

5. Nullifier ordering canonicalized (`nullifier[0] < nullifier[1]` when both real).
6. Recipient-ATA owner is bound into `pi.recipient_bind`.
7. Root-ring freshness on every proof.
