Both circuits are written in Circom 2.2 against `iden3/circomlib v2.4.0`. Sources in `circuits/`:

- [`circuits/transact.circom`](https://github.com/mmchougule/b402-solana/blob/main/circuits/transact.circom)
- [`circuits/adapt.circom`](https://github.com/mmchougule/b402-solana/blob/main/circuits/adapt.circom)
- `circuits/lib/` — sub-templates: `commitment`, `nullifier`, `spending_pub`, `merkle_verify`, `range_check`.

R1CS info files in `circuits/build/<name>.r1cs.info.txt` are generated by snarkjs at compile time and are the source of truth for constraint counts. Numbers below come from those files.

### In-circuit constraints

Per-input note (i ∈ {0, 1}):

1. Spending pub recompute: `spendingPub_i = Poseidon_2(spendKeyPubTag, spendingPriv_i)`. Proves ownership without revealing the key.
2. Commitment recompute: `commitmentIn_i = Poseidon_5(commitTag, tokenMint_i, value_i, random_i, spendingPub_i)`.
3. Merkle path verification: 26 × Poseidon_3 hashes; `mkRoot.computedRoot == merkleRoot` enforced selector-guarded by `(1 - inIsDummy_i)`.
4. Nullifier recompute: `nullifier_i = Poseidon_3(nullTag, spendingPriv_i, leafIndex_i)`, gated by `(1 - inIsDummy_i)`.
5. Range check: `value_i < 2^64`.

Per-output note (j ∈ {0, 1}):

1. `commitmentOut_j = Poseidon_5(commitTag, tokenMint_j, value_j, random_j, spendingPub_j)`, gated by `(1 - outIsDummy_j)`.
2. Range check: `value_j < 2^64`.

Global:

1. **Token consistency.** All non-dummy notes carry the same `tokenMint` when `publicAmountIn > 0` or `publicAmountOut > 0`.
2. **Balance conservation:** `Σ inValue (non-dummy) + publicAmountIn === Σ outValue (non-dummy) + publicAmountOut + relayerFee`. Enforced as an `Fr` equality (4 × 2^64 ≪ p, no wrap).
3. **Public-amount exclusivity:** `publicAmountIn * publicAmountOut === 0`.
4. **Relayer-fee binding:** `relayerFeeBind === Poseidon_3(feeBindTag, relayerFeeRecipient, relayerFee)`.
5. **Recipient binding:** `recipientBind === Poseidon_3(recipientBindTag, recipientOwnerLow, recipientOwnerHigh)`. Both halves are range-checked to 128 bits.
6. **Nullifier ordering:** `nullifier[0] < nullifier[1]` when both are non-dummy. Canonicalizes duplicate-nullifier detection on chain.

## `adapt` circuit (composability)

From `circuits/build/adapt.r1cs.info.txt`:

```
Curve:           bn-128
Wires:           17,654
Constraints:     17,582
Private inputs:  128
Public inputs:   23   (file as compiled — see Phase 9 note below)
Labels:          61,450
```

The compiled artifact reports 23 public inputs. The mainnet build of `b402-verifier-adapt` is compiled with `phase_9_dual_note`, which expects **24** public inputs — the 24th (index 23) is `outSpendingPubA`, declared in [`circuits/adapt.circom:101`](https://github.com/mmchougule/b402-solana/blob/main/circuits/adapt.circom#L101) and added to the `component main { public [...] }` block at [`adapt.circom:300-326`](https://github.com/mmchougule/b402-solana/blob/main/circuits/adapt.circom#L300-L326). Treat the 24-input shape as authoritative for current mainnet; verify against build artifacts after each recompile.

### Public inputs (24, mainnet)

The first 18 are byte-identical to `transact` for prover-side code reuse. Indices 18–22 are adapt-specific. Index 23 is the Phase 9 dual-note output spending-pub alias.

| # | Signal | Meaning |
|---|---|---|
| 0–17 | (transact prefix) | Same as transact |
| 18 | `adapterId` | `keccak256(adapter_program_id) mod p` |
| 19 | `actionHash` | `Poseidon_3(adaptBindTag, keccak(action_payload) mod p, expectedOutMint)` |
| 20 | `expectedOutValue` | Pool's minimum OUT-mint delta after the adapter CPI |
| 21 | `expectedOutMint` | OUT mint, reduced mod p |
| 22 | `adaptBindTag` | Domain tag `"b402/v1/adapt-bind"` |
| 23 | `outSpendingPubA` | `outSpendingPub[0]` alias — pool reads to recompute `commitment_b` for the excess output leaf |

The Phase 9 alias is not the full `outSpendingPub[2]` array — circom does not allow indexed elements in the `public [...]` block. The circuit constrains `outSpendingPubA === outSpendingPub[0]` ([`adapt.circom:297`](https://github.com/mmchougule/b402-solana/blob/main/circuits/adapt.circom#L297)); the dummy slot `outSpendingPub[1]` stays private.

### Differences vs transact

From [`circuits/adapt.circom`](https://github.com/mmchougule/b402-solana/blob/main/circuits/adapt.circom):

1. **Two mint bindings.** Input notes bind to `publicTokenMint` (IN mint, [`adapt.circom:235-236`](https://github.com/mmchougule/b402-solana/blob/main/circuits/adapt.circom#L235-L236)); output notes bind to `expectedOutMint` (OUT mint, [`adapt.circom:220`](https://github.com/mmchougule/b402-solana/blob/main/circuits/adapt.circom#L220)).
2. **Balance constraints diverge.** `inSum === publicAmountIn + relayerFee` and `outSum === expectedOutValue` ([`adapt.circom:254-255`](https://github.com/mmchougule/b402-solana/blob/main/circuits/adapt.circom#L254-L255)). Conservation across two mints would require an in-circuit price oracle — instead, the pool's I4 post-CPI delta check enforces the OUT-side bound.
3. **`publicAmountOut === 0`** — adapt has no public unshield path ([`adapt.circom:173`](https://github.com/mmchougule/b402-solana/blob/main/circuits/adapt.circom#L173)).
4. **Action-hash binding.** `actionHash === Poseidon_3(adaptBindTag, actionPayloadKeccakFr, expectedOutMint)` ([`adapt.circom:279-283`](https://github.com/mmchougule/b402-solana/blob/main/circuits/adapt.circom#L279-L283)). The pool recomputes `keccak256(action_payload)` mod p on chain — closes the relayer-tampered-payload attack.
5. **`adapterId` passthrough.** Pool enforces `adapterId == keccak256(adapter_program.key) mod p` on chain. In-circuit it is a passthrough so a proof for adapter X cannot replay against adapter Y (different public input → different proof bytes).
6. **Phase 9 dual-note alias.** `outSpendingPubA` lifted public so the pool can compute `commitment_b = Poseidon_5(TAG_COMMIT, outMintFr, excessFr, randomB, outSpendingPub[0])` for an excess-delta leaf when the adapter over-delivers. SDK and pool use a frozen-vector parity test ([`adapt_execute.rs:1289-1399`](https://github.com/mmchougule/b402-solana/blob/main/programs/b402-pool/src/instructions/adapt_execute.rs#L1289-L1399)) to lock the byte ordering.

## Sub-circuits

### `Commitment` (`circuits/lib/commitment.circom`)

```
out = Poseidon_5(commitTag, tokenMint, value, random, spendingPub)
```

### `Nullifier` (`circuits/lib/nullifier.circom`)

```
out = Poseidon_3(nullTag, spendingPriv, leafIndex)
```

### `SpendingPub` (`circuits/lib/spending_pub.circom`)

```
out = Poseidon_2(spendKeyPubTag, spendingPriv)
```

### `MerkleRoot` (`circuits/lib/merkle_verify.circom`)

26 levels of `Poseidon_3` with `pathBits` selectors. ~6,240 R1CS per proven input.

### `RangeCheck(N)`

Generic N-bit decomposition + sum-equals-original. Used at N=64 for amounts, N=128 for owner-pubkey halves.

## Trusted-setup status

| Circuit | Phase 2 | VK location | Mainnet |
|---|---|---|---|
| `transact` | Single-contributor (b402 team) | `circuits/build/ceremony/transact_verification_key.json` | yes (same VK as devnet) |
| `adapt` (Phase 9, 24 inputs) | Single-contributor (b402 team) | `circuits/build/ceremony/adapt_verification_key.json` | yes (same VK as devnet) |

Both VKs are baked into their respective verifier programs at build time via `circuits/scripts/vk-to-rust.mjs`. Devnet and mainnet share the same VKs today. A multi-party Phase-2 with ≥3 contributors is the planned hardening path; once run, the new verifier programs deploy and the pool rotates via `set_verifier`.

## Performance

Wall-time is dominated by snarkjs WASM proof generation (~750 ms per adapt proof on a recent laptop). On-chain Groth16 verify costs ~178k CU for transact and ~210k CU for adapt's 24-input shape. The pool reserves 1.4M CU for the whole instruction to leave headroom.

Rapidsnark (C++) would cut prover wall time to ~250 ms; v1 stays on snarkjs for portability and zero-install browser compatibility.
