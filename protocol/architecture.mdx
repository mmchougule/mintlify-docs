## Components

### Programs (`programs/`)

| Program | Role | Source |
|---|---|---|
| `b402-pool` | Shielded pool: state, custody, dispatch, adapter CPI | [`programs/b402-pool/`](https://github.com/mmchougule/b402-solana/tree/main/programs/b402-pool) |
| `b402-verifier-transact` | Groth16 verifier, 18 public inputs, VK baked at build | [`programs/b402-verifier-transact/`](https://github.com/mmchougule/b402-solana/tree/main/programs/b402-verifier-transact) |
| `b402-verifier-adapt` | Groth16 verifier; 24 public inputs under `phase_9_dual_note` (mainnet today); two entrypoints (`verify`, `verify_with_account_inputs`) | [`programs/b402-verifier-adapt/`](https://github.com/mmchougule/b402-solana/tree/main/programs/b402-verifier-adapt) |
| `b402-nullifier` | Domain-tagged fork of Light's nullifier program; CPIs Light V2 address tree to land nullifier IDs (double-spend gate) | [`programs/b402-nullifier/`](https://github.com/mmchougule/b402-solana/tree/main/programs/b402-nullifier) |
| `b402-jupiter-adapter` | Jupiter V6 swap CPI (stateless adapter) | [`programs/b402-jupiter-adapter/`](https://github.com/mmchougule/b402-solana/tree/main/programs/b402-jupiter-adapter) |
| `b402-kamino-adapter` | Kamino V2 deposit/withdraw via per-user obligation | [`programs/b402-kamino-adapter/`](https://github.com/mmchougule/b402-solana/tree/main/programs/b402-kamino-adapter) |
| `b402-mock-adapter` | Test-only adapter for delta-invariant tests | [`programs/b402-mock-adapter/`](https://github.com/mmchougule/b402-solana/tree/main/programs/b402-mock-adapter) |
| `b402-adrena-adapter` | Adrena perps (scaffolded) | [`programs/b402-adrena-adapter/`](https://github.com/mmchougule/b402-solana/tree/main/programs/b402-adrena-adapter) |
| `b402-orca-adapter` | Orca whirlpool LP (scaffolded) | [`programs/b402-orca-adapter/`](https://github.com/mmchougule/b402-solana/tree/main/programs/b402-orca-adapter) |
| `b402-jupiter-perps-adapter` | Jupiter Perps async (deferred) | [`programs/b402-jupiter-perps-adapter/`](https://github.com/mmchougule/b402-solana/tree/main/programs/b402-jupiter-perps-adapter) |

The `b402-nullifier` program is part of the trusted set, even though end users never call it directly. It is the only program permitted by the pool's `inline_cpi_nullifier` build to insert into Light Protocol's address tree V2 under the `b"b402/v1/null"` domain tag — see [`programs/b402-nullifier/src/lib.rs:46-53`](https://github.com/mmchougule/b402-solana/blob/main/programs/b402-nullifier/src/lib.rs#L46-L53). When built with `--features cpi-only`, top-level dispatch is rejected and the caller's program ID must equal `b402_pool` ([`lib.rs:96-114`](https://github.com/mmchougule/b402-solana/blob/main/programs/b402-nullifier/src/lib.rs#L96-L114)).

### Mainnet feature-flag matrix

The on-chain binaries were built with these Cargo features:

| Program | Features compiled in (mainnet, 2026-04-30+) |
|---|---|
| `b402-pool` | `prd_35_pending_inputs`, `inline_cpi_nullifier`, `phase_9_dual_note` |
| `b402-verifier-adapt` | `phase_9_dual_note` (24 public inputs) |
| `b402-verifier-transact` | none (18 public inputs is fixed) |
| `b402-nullifier` | `cpi-only` (top-level dispatch rejected; caller must be `b402_pool`) |
| `b402-kamino-adapter` | `per_user_obligation` |

A pool feature flag toggles a wire shape, an instruction surface, or a CPI shape — they are not free. SDK and pool must be upgraded together. See `programs/b402-pool/src/instructions/unshield.rs:39-41` for the field-gating example.

## Pool instruction surface

Source of truth: [`programs/b402-pool/src/lib.rs:31-159`](https://github.com/mmchougule/b402-solana/blob/main/programs/b402-pool/src/lib.rs#L31-L159).

```
init_pool                 admin, one-time; takes treasury_pubkey
add_token_config          admin, whitelist mint, set max_tvl
set_max_tvl               admin, per-mint cap update
shield                    user, deposit
transact                  user, internal transfer (no public movement)
unshield                  user, withdraw to clear ATA
adapt_execute             user, composable execution (Phase 7B inline-CPI path)
adapt_execute_v2          user, PRD-35 pending-inputs path (feature-gated)
commit_inputs             user, tx-1 of PRD-35 two-tx flow
gc_pending_inputs         admin, refund rent on stale PendingInputs PDA
register_adapter          admin, append-only adapter allowlist
set_verifier              admin, repoint verifier_transact / verifier_adapt / verifier_disclose
pause / unpause           admin, gate shields / transacts / adapts (NEVER unshield)
check_adapter_delta_mock  test-only, gated behind `test-mock` Cargo feature
```

There is no `disable_adapter`, `enable_adapter`, `queue_upgrade`, `revoke_upgrade_authority`, or `rotate_admin` instruction. Admin is a single key (`PoolConfig.admin_multisig`); multi-signer enforcement is not implemented.

## State (PDAs)

All PDAs use `b"b402/v1"` as a version prefix in seeds. Layout from [`programs/b402-pool/src/state.rs`](https://github.com/mmchougule/b402-solana/blob/main/programs/b402-pool/src/state.rs):

| PDA | Seeds | Purpose |
|---|---|---|
| `PoolConfig` | `[b"b402/v1", b"config"]` | Admin pubkey, pause flags, pinned verifier program IDs |
| `TokenConfig` | `[b"b402/v1", b"token", mint]` | Per-token: decimals, vault pubkey, enabled, `max_tvl` |
| `Vault` (token account) | `[b"b402/v1", b"vault", mint]` | Pool's custody of `mint` |
| `TreeState` | `[b"b402/v1", b"tree"]` | 128-root ring, frontier (depth 26), leaf count, zero cache (zero-copy) |
| `AdapterRegistry` | `[b"b402/v1", b"adapters"]` | Registered adapter program IDs + allowed instruction discriminators |
| `TreasuryConfig` | `[b"b402/v1", b"treasury"]` | `treasury_pubkey` for `gc_pending_inputs` rent refunds |
| `PendingInputs` | `[b"b402/v1", b"pending-inputs", spending_pub_le]` | PRD-35 per-user public-input staging (777 B at Phase 9) |

`TreeState` is fixed-size (`#[account(zero_copy)]`); see [`state.rs:53-78`](https://github.com/mmchougule/b402-solana/blob/main/programs/b402-pool/src/state.rs#L53-L78). The 128-root ring (`ROOT_HISTORY_SIZE`) is set at [`constants.rs:22`](https://github.com/mmchougule/b402-solana/blob/main/programs/b402-pool/src/constants.rs#L22). The on-chain `NullifierShard` type still exists in source ([`state.rs:109-140`](https://github.com/mmchougule/b402-solana/blob/main/programs/b402-pool/src/state.rs#L109-L140)) but is not written by the mainnet build — nullifiers land in Light V2 via the `b402-nullifier` CPI. See [Nullifier Set & Merkle Tree](/protocol/nullifier-tree).

## Cross-program invocation graph (`adapt_execute`, mainnet build)

```
relayer signs the tx
   │
   ▼
b402_pool::adapt_execute (or adapt_execute_v2)
   ├─ [PRD-35] read public inputs from pending_inputs PDA, byte-equality check
   ├─ CPI → b402_verifier_adapt::verify_with_account_inputs (Phase 9 VK, 24 pis)
   ├─ For each non-dummy input nullifier:
   │     CPI → b402_nullifier::create_nullifier
   │            └─ CPI → light_system_program (insert into Light V2 address tree)
   ├─ CPI → spl-token::transfer (in_vault → adapter_in_ta, pool PDA signs)
   │       (skipped under Path A when adapter is stateful AND in_mint is a synthetic voucher)
   ├─ CPI → spl-token::transfer (in_vault → relayer_fee_ta) when fee > 0
   ├─ snapshot out_vault.amount as `pre`
   ├─ CPI → adapter_program::execute
   │       (action_payload prefixed with viewing_pub_hash for stateful adapters)
   │       └─ CPI → Jupiter / Kamino / ... → AMM / lending programs
   ├─ reload out_vault, compute delta = post − pre
   ├─ if !stateful_voucher_mint: require delta ≥ expected_out_value (I4)
   ├─ tree_append for each non-dummy commitment_out (sol_poseidon × 26)
   ├─ [Phase 9] if delta > expected_out_value: append commitment_b for the excess
   ├─ zero pending_inputs.version (replay protection)
   └─ emit AdaptExecuted, CommitmentAppended[, ExcessNoteMinted]
```

Compute budget: 1.4M CU requested; ~660k actual on the mainnet Jupiter SolFi V2 case. Mainnet swap reference: `5aVtM9Vc3eSgakTwv8vNG2snokQCWZhc8D6vfLTYigJRTkW2eZ8Ymv5gYawD81qCfAyj3KzhvHJqvGA1znpLKUhB`.

## Path: shield → adapt_execute_v2 → unshield

```
1. user.shield                  : user → vault (USDC), tree_append commitment_in
2. user.commit_inputs           : SDK writes 24 × 32 B public inputs to PendingInputs PDA
3. user.adapt_execute_v2        : verifier reads PDA, CPI nullifier × N, CPI adapter,
                                  tree_append commitment_out (+ commitment_b if excess)
4. user.unshield                : nullifier × N, vault → recipient ATA (recipient-owner-bound)
```

The `commit_inputs` and `adapt_execute_v2` pair lifts the v0-tx 1232 B ceiling for per-user adapters by moving ~768 B of public inputs from ix data into account data. See [Pool Program](/protocol/pool-program) for the full validation order.

## Address Lookup Table (ALT)

Hosted relayer pubkey: `7f6gRiX56dMQGrPERNBKuzFsvagFTM1U4LMAAN9rsiNM`. The hosted relayer ALT pre-loads the static program/PDA accounts shared across every shielded tx (verifier, nullifier, light system program, account-compression authorities, pool PDAs, common token configs and vaults). Per-tx accounts (user ATAs, adapter routing accounts, pending-inputs PDA) ride in the `accountKeys` block and the unique-accounts list.

See [Program IDs](/reference/program-ids) for the full ID reference. Same program IDs are deployed on devnet and mainnet.
