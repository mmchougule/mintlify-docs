## Cloned mainnet account types in tests

Across the two mainnet-fork harnesses, the validator clones (with
`solana-test-validator --clone`):

- **Programs:** Jupiter v6, Kamino Lend, Kamino Farms, all AMMs
  Jupiter routes through (SolFi V2, Whirlpool, Raydium CLMM, etc.).
- **Reserves:** USDC reserve account, the corresponding
  `liquidity_supply_vault` and `collateral_supply_vault`.
- **Vaults:** every AMM pool's token account on the Jupiter route.
- **Oracles:** Pyth and Switchboard price accounts referenced by the
  Kamino reserve.
- **ATAs:** funded user ATAs for the tested mints.

Five distinct mainnet account kinds: programs, reserves, vaults,
oracles, ATAs.

## `// TODO(verify):` markers

Markers like `// TODO(verify):` flag claims in code that should be
re-checked against the upstream protocol's source or IDL before
relying on them.

| Surface | Count |
|---|---|
| `programs/` | 4 |
| Whole repo (excluding `node_modules`, `target`, `docs-site`) | 5 |

This is intentionally a **decreasing-with-time confidence metric**.
Each `TODO(verify)` is a known-soft claim that gets resolved against
mainnet bytecode at integration-test time.

## Reproduction

To rebuild every number on this page:

```bash
# Circuits (R1CS info files)
cd circuits && pnpm install && pnpm build:circuits

# BPF program builds
cargo build-sbf --tools-version v1.54 \
  --manifest-path programs/b402-pool/Cargo.toml --features test-mock
ls -la target/deploy/

# Assurance run (vectors, fuzz, invariants, bench, on-chain probe)
cd ../b402-solana-assurance
pnpm install
pnpm assure

# Mainnet-fork verifications (require Helius/Alchemy mainnet RPC)
cd ../b402-solana/examples
./ops/mainnet-fork-validator.sh --route /tmp/jup-route.json --reset    # term 1
pnpm swap-e2e-jupiter                                                    # term 2

./ops/kamino-fork-validator.sh --reset                                  # term 1
pnpm tsx kamino-adapter-fork-deposit.ts                                 # term 2
```
