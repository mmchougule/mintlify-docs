The b402 MCP server exposes the full SDK as agent-callable tools over the Model Context Protocol. Once installed, any MCP-compatible runtime — Claude Code, Cursor, or a custom agent — can perform private DeFi operations by invoking the tools described below. The server loads your Solana CLI keypair once at startup and never logs it or includes it in tool responses.

## Install

```bash
claude mcp add b402-solana -- npx -y @b402ai/solana-mcp@latest
```

This registers the server in Claude Code. The circuit artifacts (~36 MB) are bundled in the package — no separate clone or build step is needed.

<Note>
  The server defaults to **mainnet** using the keypair at `~/.config/solana/id.json`. Set `B402_CLUSTER=devnet` for risk-free testing against the deployed devnet pool.
</Note>

## Environment variables

All variables are optional. Set only the ones you need to override.

| Variable | Default | Description |
|---|---|---|
| `B402_CLUSTER` | `mainnet` | One of `mainnet`, `devnet`, or `localnet`. |
| `B402_RPC_URL` | Public cluster RPC | Private RPC endpoint. Use Helius / Triton / QuickNode in production to avoid rate limits. |
| `B402_KEYPAIR_PATH` | `~/.config/solana/id.json` | Path to the Solana keypair file used for signing. |
| `B402_CIRCUITS_ROOT` | Bundled artifacts | Override to use locally built circuit artifacts. |

```bash
claude mcp add b402-solana \
  --env B402_CLUSTER=mainnet \
  --env B402_RPC_URL=https://mainnet.helius-rpc.com/?api-key=YOUR_KEY \
  -- npx -y @b402ai/solana-mcp@latest
```

## Tools

### shield

Move SPL tokens from the wallet's Associated Token Account into a private shielded balance. This is the only tool that requires the wallet to sign a transaction on-chain.

**Input**

<ParamField body="mint" type="string" required>
  Base58-encoded SPL mint address (e.g., `EPjFWdd5AufqSSqeM2qN1xzybapC8G4wEGGkZwyTDt1v` for USDC on mainnet).
</ParamField>

<ParamField body="amount" type="string" required>
  Amount to shield as a u64 decimal string in smallest token units. For USDC, `"1000000"` = 1 USDC.
</ParamField>

**Output**

<ResponseField name="signature" type="string">
  Base58 transaction signature. Verify at `https://solscan.io/tx/<signature>`.
</ResponseField>

<ResponseField name="leafIndex" type="number">
  Merkle tree leaf index of the new shielded commitment.
</ResponseField>

<ResponseField name="depositId" type="string">
  Opaque deposit identifier — the first 16 hex characters of the commitment. Use this as a stable key to refer to the deposit in subsequent tool calls.
</ResponseField>

### unshield

Withdraw a shielded note to any public Solana address. The relayer signs and pays gas — the depositor wallet does not appear on the transaction.

<Note>
  You must have a spendable note in the requested mint. Call `shield` first, or use `holdings` to confirm available notes.
</Note>

**Input**

<ParamField body="to" type="string" required>
  Base58 public key of the recipient. The canonical Associated Token Account is derived automatically.
</ParamField>

<ParamField body="mint" type="string" required>
  Base58 mint address of the token to withdraw.
</ParamField>

<ParamField body="amount" type="string">
  Reserved for future partial-spend support. Currently the entire note value is withdrawn. Omit this field.
</ParamField>

**Output**

<ResponseField name="signature" type="string">
  Transaction signature for the unshield.
</ResponseField>

### private_swap

Atomically swap a shielded balance through a registered adapter. On mainnet, routes through Jupiter v6. On devnet, routes through the constant-rate mock adapter (constant 2x output). The relayer signs and pays gas.

<Note>
  The swap requires an exact-match deposit: a shielded note in `inMint` with a value exactly equal to `amount`. Use `shield` to create the exact deposit first, then call `private_swap`.
</Note>

**Input**

<ParamField body="inMint" type="string" required>
  Base58 mint address of the token to swap from.
</ParamField>

<ParamField body="outMint" type="string" required>
  Base58 mint address of the token to receive.
</ParamField>

<ParamField body="amount" type="string" required>
  Amount of `inMint` to swap, as a u64 decimal string.
</ParamField>

**Output**

<ResponseField name="signature" type="string">
  Transaction signature.
</ResponseField>

<ResponseField name="outAmount" type="string">
  Actual output amount received, in smallest units of `outMint`.
</ResponseField>

---

### private_lend

Atomically deposit a shielded USDC balance into a Kamino V2 reserve and mint a voucher commitment. Mainnet only — Kamino reserves do not exist on devnet.

<Warning>
  The first `private_lend` call for a given (keypair, mint) pair incurs a one-time ~0.04 SOL (~$7) fee for Kamino `UserMetadata` and `Obligation` account rent. This rent is refundable when the position is closed.
</Warning>

**Input**

<ParamField body="mint" type="string" required>
  Base58 mint address of the token to lend (currently USDC mainnet: `EPjFWdd5AufqSSqeM2qN1xzybapC8G4wEGGkZwyTDt1v`).
</ParamField>

<ParamField body="amount" type="string" required>
  Amount to lend as a u64 decimal string. Must exactly match a spendable note's value.
</ParamField>

**Output**

<ResponseField name="signature" type="string">
  Transaction signature.
</ResponseField>

---

### private_redeem

Burn a Kamino voucher commitment (minted by `private_lend`) and reshield the underlying tokens as a new private note. Mainnet only.

**Input**

<ParamField body="mint" type="string" required>
  Base58 mint address of the underlying token (e.g., USDC). The voucher mint is derived from the Kamino reserve automatically.
</ParamField>

**Output**

<ResponseField name="signature" type="string">
  Transaction signature.
</ResponseField>

<ResponseField name="amount" type="string">
  Underlying token amount reshielded, in smallest units.
</ResponseField>

### status

Returns a combined snapshot of the wallet's public balance and private shielded balance. The private side reads from the local note store; pass `refresh: true` to re-scan on-chain history first.

**Input**

<ParamField body="refresh" type="boolean">
  When `true`, scans on-chain commitment events before computing private balances. Use this after importing a wallet or switching machines. Default `false`.
</ParamField>

**Output**

<ResponseField name="user" type="string">
  Base58 public key of the connected wallet.
</ResponseField>

<ResponseField name="cluster" type="string">
  Active cluster.
</ResponseField>

<ResponseField name="public" type="object">
  <Expandable title="public balance fields">
    <ResponseField name="sol" type="object">
      SOL balance: `lamports` (string) and `ui` (formatted string).
    </ResponseField>
    <ResponseField name="tokens" type="array">
      SPL token accounts with `mint`, `amount`, `uiAmount`, `decimals`, and `tokenAccount`.
    </ResponseField>
  </Expandable>
</ResponseField>

<ResponseField name="private" type="object">
  <Expandable title="private balance fields">
    <ResponseField name="totalDeposits" type="number">
      Total count of spendable notes across all mints.
    </ResponseField>
    <ResponseField name="balances" type="array">
      Per-mint aggregates: `mint`, `amount`, `depositCount`.
    </ResponseField>
  </Expandable>
</ResponseField>

### holdings

Returns a per-deposit list of private holdings. Each entry is one spendable note that can be spent independently. Use this when you need deposit-level visibility rather than aggregate totals.

**Input**

<ParamField body="mint" type="string">
  Filter to notes for a specific mint (base58 address). Omit to list holdings across all mints.
</ParamField>

**Output**

<ResponseField name="holdings" type="array">
  <Expandable title="holding entry fields">
    <ResponseField name="id" type="string">
      Opaque deposit ID (first 16 hex chars of the commitment). Stable; does not expose any spendable data.
    </ResponseField>
    <ResponseField name="mint" type="string">
      Base58 mint address of this deposit.
    </ResponseField>
    <ResponseField name="amount" type="string">
      Note value in smallest units.
    </ResponseField>
  </Expandable>
</ResponseField>

### balance

Returns aggregate private (shielded) balances grouped by mint. Reads from the local note store — no RPC call unless `refresh` is also set via `status`.

**Input**

<ParamField body="mint" type="string">
  Filter to a specific mint (base58 address). Omit to return balances for all mints.
</ParamField>

**Output**

<ResponseField name="balances" type="object">
  Map of mint address to balance info: `amount` (string, in smallest units) and `depositCount` (number of contributing notes).
</ResponseField>

---

### quote_swap

Fetch a Jupiter swap quote for an intended `private_swap`. Returns the expected output amount, slippage floor, and price impact. Mainnet routes only — Jupiter does not index devnet liquidity.

<Warning>
  A `RouteNotFound` error on devnet is expected: Jupiter only indexes mainnet liquidity. Use this tool on mainnet only.
</Warning>

**Input**

<ParamField body="inMint" type="string" required>
  Base58 mint address of the input token.
</ParamField>

<ParamField body="outMint" type="string" required>
  Base58 mint address of the output token.
</ParamField>

<ParamField body="amount" type="string" required>
  Input amount as a u64 decimal string.
</ParamField>

**Output**

<ResponseField name="expectedOut" type="string">
  Estimated output amount in smallest units of `outMint`.
</ResponseField>

<ResponseField name="slippage" type="number">
  Slippage tolerance in basis points used for the quote.
</ResponseField>

<ResponseField name="priceImpact" type="string">
  Estimated price impact as a percentage string.
</ResponseField>

<ResponseField name="routeHops" type="number">
  Number of hops in the Jupiter route.
</ResponseField>

### watch_incoming

Cursor-based poll for newly arrived private deposits since the last call. Pass the `cursor` from a previous response to see only new arrivals. Omit the cursor (or pass `undefined`) on the first call to read all notes from the beginning.

The SDK's live note subscription pushes new arrivals into the in-memory store as they land on-chain, so polling every few seconds gives a near-real-time stream of incoming deposits.

**Input**

<ParamField body="cursor" type="string">
  Opaque cursor token returned by a previous `watch_incoming` call. Omit to read from the beginning.
</ParamField>

**Output**

<ResponseField name="incoming" type="array">
  <Expandable title="note entry fields">
    <ResponseField name="id" type="string">
      Opaque deposit ID.
    </ResponseField>
    <ResponseField name="mint" type="string">
      Base58 mint address.
    </ResponseField>
    <ResponseField name="amount" type="string">
      Note value in smallest units.
    </ResponseField>
    <ResponseField name="receivedAt" type="number">
      Unix timestamp (ms) when the note was observed locally.
    </ResponseField>
  </Expandable>
</ResponseField>

<ResponseField name="cursor" type="string">
  Opaque cursor for the next poll. Pass this back unchanged on the next call.
</ResponseField>
