The `B402Solana` class is the primary integration surface for the b402 SDK. It wraps keypair management, circuit artifact loading, Merkle proof construction, and relayer routing internally so you can perform private DeFi operations in a handful of lines. All methods call `ready()` implicitly — but calling it explicitly before your first operation avoids the one-time initialization latency on that first call.

## Initialization

### Constructor

```typescript
import { B402Solana } from '@b402ai/solana';
import { Keypair } from '@solana/web3.js';

const b402 = new B402Solana({
  cluster: 'mainnet',
  keypair: yourKeypair,
  rpcUrl: process.env.RPC_URL,
});
```

<ParamField body="cluster" type='"mainnet" | "devnet" | "localnet"' required>
  The Solana cluster to connect to. Cluster-appropriate defaults are applied for the relayer URL, indexer URL, and ALT address.
</ParamField>

<ParamField body="keypair" type="Keypair">
  Solana `Keypair` used for signing. Pass this **or** `signer`, not both. Use this path for Node.js scripts, MCP servers, and test environments.
</ParamField>

<ParamField body="signer" type="B402Signer">
  Custom signer abstraction — use `WalletAdapterSigner.fromAdapter(adapter)` for browser wallet integrations. Pass this **or** `keypair`, not both.
</ParamField>

<ParamField body="rpcUrl" type="string">
  Solana RPC endpoint. Defaults to the public cluster endpoint. Use a private RPC (Helius, Triton, QuickNode) in production to avoid rate limiting.
</ParamField>

<ParamField body="relayerHttpUrl" type="string">
  URL of an HTTP relayer. When set, `unshield`, `swap`, `lend`, and `redeem` route through the relayer — it signs and pays gas, so your wallet does not appear on those transactions. Defaults to the b402 hosted relayer for `mainnet` and `devnet`.
</ParamField>

<ParamField body="relayerApiKey" type="string">
  API key forwarded as `Authorization: Bearer <key>` to the relayer. A public-tier key is bundled as a default; replace it for higher rate limits.
</ParamField>

<ParamField body="proverArtifacts" type="ProverArtifacts">
  Paths to the transact circuit `wasm` and `zkey` files. Required if `prover` is not supplied. In the MCP server and zero-config SDK path, artifacts are fetched and cached automatically on the first `ready()` call.
</ParamField>

<ParamField body="adaptProverArtifacts" type="ProverArtifacts">
  Paths to the adapt circuit `wasm` and `zkey` files. Required for `privateSwap`, `lend`, and `redeem`.
</ParamField>

<ParamField body="notesPersistDir" type="string">
  Directory for on-disk note persistence. Defaults to `~/.b402ai/notes/<cluster>/`. Pass an empty string to disable persistence (e.g., in ephemeral CI environments). Notes are stored as plaintext JSON — same threat model as the Solana keypair file they were derived from.
</ParamField>

<ParamField body="indexerUrl" type="string">
  URL of the b402 indexer API. Enables Merkle proof fetching for any leaf (not just the most recent), which allows spending notes in any order across multiple devices. Defaults to the hosted indexer on mainnet. Pass an empty string to disable.
</ParamField>

<ParamField body="inlineCpiNullifier" type="boolean">
  When `true`, nullifier instructions are sent as an inline CPI instead of a sibling instruction, saving ~150 wire bytes per nullifier. Defaults to `true` for mainnet and devnet (required by the deployed pool), `false` for localnet.
</ParamField>

<ParamField body="programIds" type="Partial<typeof PROGRAM_IDS>">
  Override individual program IDs. Use this for localnet testing against custom-deployed programs.
</ParamField>

## Core methods

### b402.ready()

```typescript
await b402.ready(): Promise<void>
```

Lazily initializes the internal wallet, note store, and relayer health check. Safe to call multiple times — subsequent calls are no-ops. If `relayerHttpUrl` is configured, `ready()` performs a `/health` check and validates that the relayer is wired to the expected pool program.

Call `ready()` explicitly before your first operation to front-load initialization latency. All other methods call it internally, so it is never strictly required.

---

### b402.shield()

```typescript
await b402.shield(req: ShieldRequest): Promise<ShieldResult>
```

Moves SPL tokens from your wallet's Associated Token Account into the b402 shielded pool. This is the only operation that requires your wallet to sign — after shielding, subsequent swaps, lends, and unshields go through the relayer.

**Parameters**

<ParamField body="mint" type="PublicKey" required>
  The SPL mint of the token to shield. Must be whitelisted in the pool's `TokenConfig`.
</ParamField>

<ParamField body="amount" type="bigint" required>
  Amount in raw token units (smallest denomination). For USDC with 6 decimals, `1_000_000n` = 1 USDC.
</ParamField>

<ParamField body="omitEncryptedNotes" type="boolean">
  When `true`, skips publishing the ~120-byte encrypted note ciphertext on-chain. Default `false` — the ciphertext is published so notes are recoverable from chain history alone. Set to `true` only for ephemeral sessions where chain-side recovery is not needed.
</ParamField>

**Return value**

<ResponseField name="signature" type="string">
  Base58-encoded transaction signature for the shield transaction.
</ResponseField>

<ResponseField name="note" type="SpendableNote">
  The shielded note created by this deposit. Stored in the local `NoteStore` automatically.
</ResponseField>

<ResponseField name="leafIndex" type="number">
  Position of the note's commitment in the on-chain Merkle tree. Used for Merkle proof construction on spend.
</ResponseField>

<CodeGroup>
```typescript Shield USDC
const USDC = new PublicKey('EPjFWdd5AufqSSqeM2qN1xzybapC8G4wEGGkZwyTDt1v');

const result = await b402.shield({
  mint: USDC,
  amount: 1_000_000n, // 1 USDC
});
console.log('Shield tx:', result.signature);
console.log('Leaf index:', result.note.leafIndex);
```
</CodeGroup>

### b402.unshield()

```typescript
await b402.unshield(req: UnshieldRequest): Promise<UnshieldResult>
```

Withdraws a shielded note to a public Solana address. The relayer signs and pays gas — your wallet does not appear on the transaction. By default, the most recently shielded note from this session is spent. Pass `note` and `mint` to spend any specific note.

**Parameters**

<ParamField body="to" type="PublicKey" required>
  Owner of the destination token account. The canonical Associated Token Account for `mint` is derived automatically.
</ParamField>

<ParamField body="mint" type="PublicKey">
  SPL mint of the note to spend. Required when `note` is supplied. If neither `note` nor `mint` is provided, the mint from the last `shield()` call is used.
</ParamField>

<ParamField body="note" type="SpendableNote">
  Explicit note to spend. Defaults to the most-recently-shielded note in the given `mint`. Pass this to spend a specific deposit (e.g., from a persisted note store).
</ParamField>

<ParamField body="recipientAta" type="PublicKey">
  Override the destination Associated Token Account. Defaults to the canonical ATA of `to` for `mint`. The SDK creates the ATA if it does not exist.
</ParamField>

<ParamField body="inlineCpiNullifier" type="boolean">
  Per-call override of `B402SolanaConfig.inlineCpiNullifier`. See constructor docs.
</ParamField>

**Return value**

<ResponseField name="signature" type="string">
  Base58-encoded transaction signature for the unshield transaction.
</ResponseField>

<CodeGroup>
```typescript Unshield to a recipient
const result = await b402.unshield({
  to: recipientPublicKey,
  mint: USDC,
});
console.log('Unshield tx:', result.signature);
```

```typescript Unshield a specific note
const result = await b402.unshield({
  to: recipientPublicKey,
  mint: USDC,
  note: myNote,
});
```
</CodeGroup>

### b402.swap()

```typescript
await b402.swap(req): Promise<PrivateSwapResult>
```

High-level private swap with auto-routing via Jupiter. Burns a shielded note in `inMint`, routes through Jupiter (Phoenix-direct on mainnet), and reshields the proceeds into a new note in `outMint` — all in one atomic transaction. On devnet, falls through to the constant-rate mock adapter. The relayer signs and pays gas.

**Note value constraint:** you must hold a shielded note in `inMint` with a value exactly equal to `amount`. Use `b402.shield()` to create an exact-match deposit first.

**Parameters**

<ParamField body="inMint" type="PublicKey" required>
  SPL mint of the token to swap from. A shielded note in this mint with value equal to `amount` must exist in your note store.
</ParamField>

<ParamField body="outMint" type="PublicKey" required>
  SPL mint of the token to receive. The output is reshielded as a new note.
</ParamField>

<ParamField body="amount" type="bigint" required>
  Amount of `inMint` to swap, in smallest units. Must exactly match a spendable note's value.
</ParamField>

<ParamField body="slippageBps" type="number">
  Slippage tolerance in basis points. Default `30` (0.3%). The Jupiter quote's `otherAmountThreshold` at this slippage is used as the `expectedOut` floor bound into the proof.
</ParamField>

<ParamField body="expectedOut" type="bigint">
  Override the expected output amount. Default: derived from the Jupiter quote with `slippageBps` applied.
</ParamField>

<ParamField body="dexes" type="string">
  DEX allowlist for Jupiter routing. Default `"Phoenix"` (smallest account count, fits within the 1,232-byte transaction cap reliably).
</ParamField>

**Return value**

<ResponseField name="signature" type="string">
  Transaction signature for the swap.
</ResponseField>

<ResponseField name="outNote" type="SpendableNote">
  New shielded note in `outMint`, bound to the proof's `expectedOut` value.
</ResponseField>

<ResponseField name="outAmount" type="bigint">
  Actual output vault delta observed on-chain post-swap.
</ResponseField>

<ResponseField name="excessNote" type="SpendableNote">
  Present when `outAmount > expectedOut` (Phase 9 dual-note). The excess is minted as a second commitment. The two notes sum to exactly `outAmount`.
</ResponseField>

<CodeGroup>
```typescript Private USDC → SOL swap
const USDC = new PublicKey('EPjFWdd5AufqSSqeM2qN1xzybapC8G4wEGGkZwyTDt1v');
const SOL  = new PublicKey('So11111111111111111111111111111111111111112');

// Shield first (exact match required)
await b402.shield({ mint: USDC, amount: 1_000_000n });

// Swap
const { signature, outNote, outAmount } = await b402.swap({
  inMint: USDC,
  outMint: SOL,
  amount: 1_000_000n,
});
console.log('Swap tx:', signature);
console.log('Received (lamports):', outAmount.toString());
```
</CodeGroup>

### b402.lend()

```typescript
await b402.lend(req): Promise<PrivateSwapResult>
```

Atomically deposits a shielded note into a Kamino V2 reserve and mints a voucher commitment in the same transaction. Auto-discovers the deepest reserve for `mint` across all Kamino markets. Mainnet only.

**First call per (viewing key, mint)** incurs a one-time ~0.04 SOL fee for Kamino `UserMetadata` + `Obligation` account rent. This rent is refundable via Kamino's `close_obligation` when the position is empty.

**Parameters**

<ParamField body="mint" type="PublicKey" required>
  Underlying SPL mint to lend (e.g., USDC). A shielded note in this mint must exist.
</ParamField>

<ParamField body="amount" type="bigint" required>
  Amount to lend, in smallest units. Must exactly match a spendable note's value.
</ParamField>

<ParamField body="market" type="PublicKey">
  Pin a specific Kamino `LendingMarket` address. If unset, the market with the deepest available liquidity for `mint` is selected automatically.
</ParamField>

**Return value**

<ResponseField name="signature" type="string">
  Transaction signature.
</ResponseField>

<ResponseField name="outNote" type="SpendableNote">
  Voucher note (Kamino collateral token commitment) minted by this deposit.
</ResponseField>

<ResponseField name="outAmount" type="bigint">
  Collateral token amount minted on-chain.
</ResponseField>

<CodeGroup>
```typescript Private USDC lend
const result = await b402.lend({
  mint: USDC,
  amount: 1_000_000n, // 1 USDC
});
console.log('Lend tx:', result.signature);
```
</CodeGroup>

```typescript
await b402.redeem(req): Promise<PrivateSwapResult>
```

**Parameters**

</ParamField>

<ParamField body="market" type="PublicKey">
  Pin a specific Kamino `LendingMarket`. Must match the market used at lend time.
</ParamField>

<ParamField body="leafIndex" type="number">
  Merkle leaf index of the voucher note to burn. Defaults to the most recently minted voucher in the local note store.
</ParamField>

**Return value**

<ResponseField name="signature" type="string">
  Transaction signature.
</ResponseField>

<ResponseField name="outNote" type="SpendableNote">
  New shielded note for the redeemed underlying tokens.
</ResponseField>

<ResponseField name="outAmount" type="bigint">
  Underlying token amount received on-chain.
</ResponseField>

<CodeGroup>
```typescript Redeem a Kamino voucher

  mint: USDC,

console.log('Redeem tx:', result.signature);
```
</CodeGroup>

### b402.balance()

```typescript
await b402.balance(opts?): Promise<{ balances: Array<{ mint: string; amount: string; depositCount: number }> }>
```

Returns aggregate private (shielded) balances grouped by mint. Reads from the local `NoteStore` — no RPC call. Use `refresh: true` to re-scan on-chain history first if the local store may be stale (e.g., after using a different machine).

<ParamField body="mint" type="PublicKey">
  Filter to a single mint. When provided, the mint's base58 address is used as the key in the response.
</ParamField>

<ParamField body="refresh" type="boolean">
  Re-scan on-chain commitment events before computing aggregates. Default `false`.
</ParamField>

**Return value**

<ResponseField name="balances" type="array">
  <Expandable title="balance entry fields">
    <ResponseField name="mint" type="string">
      Base58 mint address, or an opaque `unknown:<12hex>` label for mints not yet seen by this SDK instance.
    </ResponseField>
    <ResponseField name="amount" type="string">
      Total shielded amount across all spendable notes for this mint, in smallest units.
    </ResponseField>
    <ResponseField name="depositCount" type="number">
      Number of individual spendable notes contributing to this balance.
    </ResponseField>
  </Expandable>
</ResponseField>

<CodeGroup>
```typescript Read aggregate private balance
const { balances } = await b402.balance({ mint: USDC });

```
</CodeGroup>

### b402.privateSwap()

```typescript
await b402.privateSwap(req: PrivateSwapRequest): Promise<PrivateSwapResult>
```

Low-level private swap. You supply the adapter wiring directly — adapter program ID, scratch token accounts, ALT, instruction data, remaining accounts, and a Light Protocol Photon RPC client. For most use cases, prefer `b402.swap()` which handles all of this automatically for Jupiter on mainnet.

Use `privateSwap()` when integrating a custom adapter or when you need to pass a pre-fetched Jupiter quote's instruction bytes directly.

<ParamField body="inMint" type="PublicKey" required>
  SPL mint of the input token.
</ParamField>

<ParamField body="outMint" type="PublicKey" required>
  SPL mint of the output token.
</ParamField>

<ParamField body="amount" type="bigint" required>
  Amount of `inMint` to swap. Must be greater than zero and must exactly match a spendable note's value.
</ParamField>

<ParamField body="adapterProgramId" type="PublicKey">
  Adapter program to route through. Defaults to `programIds.b402JupiterAdapter`.
</ParamField>

<ParamField body="adapterInTa" type="PublicKey" required>
  Adapter's scratch ATA for `inMint`, owned by the adapter PDA.
</ParamField>

<ParamField body="adapterOutTa" type="PublicKey" required>
  Adapter's scratch ATA for `outMint`, owned by the adapter PDA.
</ParamField>

<ParamField body="alt" type="PublicKey" required>
  Address Lookup Table to compress account metas. Required to fit within Solana's 1,232-byte transaction cap.
</ParamField>

<ParamField body="photonRpc" type="unknown" required>
  Light Protocol Photon RPC client (`createRpc(...)` from `@lightprotocol/stateless.js`). Used to fetch the nullifier non-inclusion proof before submitting.
</ParamField>

<ParamField body="expectedOut" type="bigint">
  Expected output amount in smallest units of `outMint`. Bound into the ZK proof. The pool enforces `actualOut >= expectedOut`.
</ParamField>

<ParamField body="adapterIxData" type="Uint8Array">
  Raw adapter instruction data. Defaults to the mock-adapter shape (discriminator + amount + expectedOut + action_payload vec).
</ParamField>

<ParamField body="actionPayload" type="Uint8Array">
  Borsh-encoded adapter action. Defaults to 8 zero bytes (mock adapter).
</ParamField>

<ParamField body="remainingAccounts" type="array">
  Adapter-specific remaining accounts forwarded to the adapter CPI verbatim.
</ParamField>

<ParamField body="note" type="SpendableNote">
  Override which note to spend. Defaults to the most-recently-shielded note in `inMint` whose value matches `amount`.
</ParamField>

<ParamField body="alts" type="PublicKey[]">
  Additional Address Lookup Tables (up to 3 extra; Solana allows 4 ALTs per transaction). Use for adapters whose route references a published ALT (e.g., Jupiter's per-quote ALT).
</ParamField>

**Return value**

<ResponseField name="signature" type="string">
  Transaction signature.
</ResponseField>

<ResponseField name="outNote" type="SpendableNote">
  New shielded note in `outMint`.
</ResponseField>

<ResponseField name="outAmount" type="bigint">
  Actual output vault delta observed on-chain.
</ResponseField>

<ResponseField name="excessNote" type="SpendableNote">
  Present when `outAmount > expectedOut`. The excess is minted as a second commitment (Phase 9 dual-note).
</ResponseField>

### fetchJupiterRoute

```typescript
import { fetchJupiterRoute } from '@b402ai/solana';

const route = await fetchJupiterRoute({
  inMint,
  outMint,
  amount,
  slippageBps: 30,
  userPublicKey: adapterAuthority,
});
```

Fetches a Jupiter Lite API quote and swap instruction. Used internally by `b402.swap()`. Export this when you want to pre-fetch the quote, inspect the route, or pass the instruction data directly to `privateSwap()`.

<ResponseField name="quote" type="object">
  Jupiter quote response including `outAmount`, `otherAmountThreshold`, `slippageBps`, and `priceImpactPct`.
</ResponseField>

<ResponseField name="swap" type="object">
  Jupiter swap response including `swapInstruction` (instruction data + account metas) and `addressLookupTableAddresses`.
</ResponseField>

### discoverKaminoReserves

```typescript
import { discoverKaminoReserves } from '@b402ai/solana';

const reserves = await discoverKaminoReserves(connection);
```

Scans all Kamino `LendingMarket` accounts on-chain and returns all reserves with their mint, available amount, total supply, and collateral mint. No static reserve map — this always reflects current on-chain state across all 182+ Kamino markets.

<ResponseField name="" type="DiscoveredReserve[]">
  Array of discovered reserves, each with `address`, `market`, `mint`, `collateralMint`, `availableAmount`, and `totalSupply`.
</ResponseField>

### Scanner

```typescript
import { Scanner } from '@b402ai/solana';

const scanner = new Scanner({ connection, poolProgramId, wallet });
await scanner.scan();
```

Low-level on-chain scanner that parses commitment events emitted by the pool program. Used internally by `NoteStore.backfill()`. Useful for building custom indexers or cross-device note recovery without running the full SDK.

### NoteStore

```typescript
import { NoteStore } from '@b402ai/solana';
```

In-memory (optionally persisted) store for `SpendableNote` objects. Tracks which notes have been spent, filters by mint, and supports cursor-based scanning for new arrivals. The `B402Solana` class creates and manages a `NoteStore` instance internally; export this for advanced callers building custom tooling.

### B402Indexer

```typescript
import { B402Indexer } from '@b402ai/solana';

const indexer = new B402Indexer({ url, connection, poolProgramId });
const proof = await indexer.proveLeaf(leafIndex);
```

Client for the b402 indexer REST API. Fetches Merkle inclusion proofs for arbitrary leaf indices (not just the rightmost), enabling spend of any note in any order. The `B402Solana` class uses this automatically when `indexerUrl` is configured. Export for custom spend flows or double-spend pre-checks.

<ResponseField name="proveLeaf(leafIndex)" type="Promise<MerkleProof>">
  Fetch a Merkle proof for the given leaf index. The SDK verifies the returned root against the on-chain `TreeState` before using the proof.
</ResponseField>

<ResponseField name="isSpent(nullifierHash)" type="Promise<boolean>">
  Check whether a nullifier is already in the on-chain nullifier set. Used by `privateSwap()` to filter already-spent notes before building the proof.
</ResponseField>
